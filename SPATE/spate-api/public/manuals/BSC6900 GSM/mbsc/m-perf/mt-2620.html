Number of Successful Uplink Assignments

Measurement Counters

ID Counter Description
1279170418 SUCC.UPLINK.ASS A9502:Number of Successful Uplink Assignments
1279170420 SUCC.UPLINK.IMM.ASS A9504:Number of Successful Uplink PS Immediate Assignments
1279170422 SUCC.PACCH.UPLINK.ASS A9506:Number of Successful Uplink Assignments on PACCH

Description

These three measurements provide the number of successful uplink assignments in a granularity period. Successful assignment means that the BSC receives the uplink data block or single block. This measurement indicates the GPRS QoS of a cell. A low measurement value indicates that the radio environment, network parameter configuration, network resource configuration, or network equipment is not satisfactory, and thus requires optimization.

Measurement point

An uplink assignment is successful in any of the following conditions:

One phase access

If the BSC receives uplink data blocks from an MS after sending a packet immediate assignment message, the MS successfully receives the uplink assignment message and the uplink assignment is successful.

Figure 1 shows the uplink assignment procedure during one phase access initiated by the BSC. Each time the BSC receives the first uplink data block from the MS after sending a packet immediate assignment message (see measurement point A), this counter is incremented by one.

Figure 1 One phase access

Single-block immediate assignment

If the BSC receives a PACKET RESOURCE REQUEST message or measurement reports from an MS after sending an immediate assignment message, the MS successfully receives the single-block assignment message and the uplink assignment is successful. Figure 2 shows the BSC-initiated single-block immediate assignment procedure. Each time the BSC receives a PACKET RESOURCE REQUEST message from an MS (see measurement point A), this counter is incremented by one.

Figure 2 Single-block immediate assignment

MS-initiated packet resource request

If the BSC receives uplink data blocks from an MS after sending a packet uplink assignment message, the MS successfully receives the uplink assignment message and the uplink assignment is successful.

Figure 3 shows the packet uplink assignment procedure initiated by the BSC. Each time the BSC receives an uplink data block from the MS after sending a packet uplink assignment message (see measurement point A), this counter is incremented by one.

Figure 3 MS-initiated packet resource request

Packet resource request initiated by sending a PACKET DOWNLINK ACK/NACK message containing Channel Request Description

If the BSC receives uplink data blocks from an MS on the allocated channel after sending a PACKET UPLINK ASSIGNMENT message in response to the PACKET DOWNLINK ACK/NACK message containing Channel Request Description from the MS, the MS successfully receives the uplink assignment message and the uplink assignment is successful. Figure 4 shows the procedure for requesting packet resources in the PACKET DOWNLINK ACK/NACK message containing CHANNEL REQUEST. Each time the BSC receives an uplink data block after sending a PACKET UPLINK ASSIGNMENT message (see measurement point A), this counter is incremented by one.

Figure 4 Packet resource request initiated by sending a PACKET DOWNLINK ACK/NACK message containing Channel Request Description
