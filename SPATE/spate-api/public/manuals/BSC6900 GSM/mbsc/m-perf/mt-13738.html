Number of Failed Downlink EGPRS TBF Establishments due to MS No Response (Overlaid Subcell)

Measurement Counters

ID Counter Description
1282432186 DOWN.EGPRS.TBF.ESTB.FAIL.MS.NO.RESP.OVERLAY A9337:Number of Failed Downlink EGPRS TBF Establishments due to MS No Response (Overlaid Subcell)

Description

This counter provides the number of failed downlink EGPRS TBF establishments due to no response from the MS in the overlaid subcell of a concentric cell in a measurement period. If the value of this counter is high, the radio network quality of the overlaid subcell is bad.

Measurement point

The downlink EGPRS TBF establishment failures due to no response from the MS involve the following conditions:

Failed downlink EGPRS TBF establishment initiated on the CCCH due to MS no response

In the downlink EGPRS TBF establishment procedure, the BSC sends a POLLING message to the MS to obtain the TA value. The BSC also reserves blocks for the MS to respond with an assignment acknowledgement message. If the BSC does not receive a PACKET CONTROL ACKNOWLEDGEMENT message from the MS in the reserved blocks, the BSC sends downlink immediate assignment messages repeatedly until the maximum retry time is exceeded.

Figure 1 shows the procedure of downlink EGPRS TBF establishment failure due to no response from the MS. This counter is incremented by one when the number of BSC retry times in the overlaid subcell exceeds the maximum retry time (see measurement point A).

Figure 1 Failed downlink EGPRS TBF establishment initiated on the CCCH due to no response from the MS

Failed downlink EGPRS TBF establishment initiated on the PACCH due to MS no response

The BSC establishes a downlink EGPRS TBF in two ways:

1. The BSC initiates a new downlink EGPRS TBF establishment request during the transmission of the current uplink EGPRS TBF or the release of the current downlink EGPRS TBF.

2. The BSC reassigns downlink resources for the EGPRS TBF during the transmission of the current downlink EGPRS TBF.

The BSC sends a PACKET DOWNLINK ASSIGNMENT message and reserves blocks for the MS to respond with an assignment acknowledgement message. If the BSC does not receive a PACKET CONTROL ACKNOWLEDGEMENT message from the MS in the reserved blocks, the BSC sends downlink packet assignment messages repeatedly until the maximum retry time is exceeded.

Figure 2 shows the procedure of downlink EGPRS TBF establishment failure due to no response from the MS. This counter is incremented by one when the number of retry times exceeds the maximum retry time (see measurement point A).

Figure 2 Failed downlink EGPRS TBF establishment initiated on the PACCH due to no response from the MS
