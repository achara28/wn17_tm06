Call Drops on TCHF Due to BTS local switching Start Failure (Failures on the BTS Side)

Measurement Counters

ID Counter Description
1278089486 CELL.CALL.DROPS.BTSLOOP.FAIL.BTS.FAIL.TCHF M3121a:Call Drops on TCHF Due to BTS local switching Start Failure (Failures on the BTS Side)

Description

After the call is put through or the incoming BSC handover for the speech service is complete, the calling module initiating local switching start at one end sends a local switching request message to the calling module at the peer end within the BSC during the BTS local switching start. The call processing module at the peer end sends a BTS local switching request message to the BTS. The BTS speech local switching in the cell at the peer end fails because of various reasons. The BTS returns a BTS local switching acknowledgement message to the call processing module with the cause value of BTS local switching failure. Then, the call processing module at the peer end responds to the call processing module at the end initiating local switching start with a local switching acknowledgement message with the cause value of failures on the BTS side. The BSC regards it as a call drop and releases the call directly.

This counter provides the number of call drops on TCHF due to BTS local switching start failure (failures on the BTS side).

Measurement point

After the BSC receives the CONNECT ACK message from the MSC in the MS terminated call procedure, as shown in Figure 1 (A1), or after the BSC receives the HANDOVER CMP message from the MS in the incoming BSC handover procedure for the speech service, as shown in Figure 2 (B1), the BSC selects the BSS local switch policy, determines the local switch start conditions, and performs local switch call detection. If all the local switch conditions are met, the BTS local switching start procedure is started. Within the BSC, the call processing module initiating local switching start at one end sends a local switching request message to the call processing module at the peer end. The call processing module at the peer end sends a BTS local switching request message to the BTS. The BTS speech local switching in the cell at the peer end fails because of various reasons. The BTS returns a BTS local switching acknowledgement message to the call processing module with the cause value of local switching failure on the BTS side. Then, the call processing module at the peer end responds to the call processing module at the end initiating local switching start with a local switching acknowledgement message with the cause value of failures on the BTS side. The BSC regards it as a call drop and releases the call directly. If the channel that carries the call during the call drop is a TCHF, this counter is measured.
