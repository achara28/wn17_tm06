Number of Downlink Assignments

Measurement Counters

ID Counter Description
1279170423 DOWN.ASS A9507:Number of Downlink Assignments
1279170425 DOWN.IMM.ASS A9509:Number of Downlink PS Immediate Assignments
1279170427 PACCH.DOWN.ASS A9511:Number of Downlink Assignments on PACCH

Description

These three measurements provide the number of downlink assignment messages in a granularity period, which is the total number of downlink packet immediate assignments and downlink assignments on the PACCH.

Measurement point

The downlink assignment involves the following aspects:

Downlink packet immediate assignment on the CCCH

When the BSC receives the downlink data packets from the SGSN, and no uplink or downlink TBF exists, the BSC needs to establish the downlink TBF by sending downlink packet immediate assignment messages on the CCCH. Figure 1 shows the downlink packet immediate assignment procedure initiated by the BSC. Each time the BSC sends an IMMEDIATE ASSIGNMENT message (see measurement point A), the counter is incremented by one.

Figure 1 Downlink packet immediate assignment

Downlink assignment on the PACCH

When the BSC receives the downlink data packets from the SGSN, and the BSC detects that an uplink TBF has been established for the MS or the downlink TBF has been properly released but the timer T3193 has not been expired, the BSC needs to establish the downlink TBF by sending downlink packet immediate assignment messages on the PACCH. Figure 2 shows the packet downlink assignment procedure initiated by the BSC on the PACCH. Each time the BSC sends a PACKET DOWNLINK ASSIGNMENT message (see measurement point A), the counter is incremented by one.

Figure 2 Packet downlink assignment on the PACCH
