Description

This counter provides the total number of valid uplink RLC data blocks using CS1-CS4 coding schemes in a cell in a measurement period. The higher the proportion of valid data blocks, the fewer the retransmission times.

The following table describes the meaning of each counter.

Counter Description
UP.CS1.RLC.BLK.VALID.TOTAL Number of Valid Uplink RLC Data Blocks Using CS1
UP.CS2.RLC.BLK.VALID.TOTAL Number of Valid Uplink RLC Data Blocks Using CS2
UP.CS3.RLC.BLK.VALID.TOTAL Number of Valid Uplink RLC Data Blocks Using CS3
UP.CS4.RLC.BLK.VALID.TOTAL Number of Valid Uplink RLC Data Blocks Using CS4

Measurement point

This counter is incremented by one when a GPRS RLC data block received by the BSC from the MS is defined as a valid uplink data block in a measurement period. The data block is defined as valid when all the following conditions are met:

1.The data block uses a CS1 coding scheme.

2.The block sequential number of the data block is within the range of an RLC receive window.

3.The data block receiving state in the receive state array V(N) on the BSC side is NACKED, UNRECEIVED, or RECEIVED.

The measurement point of data block using a CS2, CS3, or CS4 coding scheme is the same as that using the CS1 coding scheme.
