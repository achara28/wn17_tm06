Number of Uplink GPRS TBF Normal Releases

Measurement Counters

ID Counter Description
1279173421 UP.GPRS.TBF.NORMAL.REL A9005:Number of Uplink GPRS TBF Normal Releases

Description

This measurement provides the number of normal uplink GPRS TBF releases in a granularity period. A low measurement value indicates that the radio environment, network parameter configuration, or network resource configuration is not satisfactory, and thus requires optimization. Further analysis should be made based on the causes of GPRS TBF abnormal release.

Measurement point

During the countdown procedure of uplink GPRS TBF, the BSC sends a PACKET UPLINK ACK/NACK (FAI = 1) message upon receiving the uplink RLC data blocks (CV = 0). The purpose is to initiate the release procedure of the uplink GPRS TBF. If the BSC receives the PACKET CONTROL ACKNOWLEDGEMENT message in the blocks assigned to the MS, then the uplink GPRS TBF is normally released.

Figure 1 shows the procedure of uplink GPRS TBF normal release. The BSC instructs the MS to release the uplink GPRS TBF by sending the PACKET UPLINK ACK/NACK (FAI = 1) message. When the BSC receives the PACKET CONTROL ACKNOWLEDGEMENT message from the MS (see measurement point A), the counter is incremented by one.
