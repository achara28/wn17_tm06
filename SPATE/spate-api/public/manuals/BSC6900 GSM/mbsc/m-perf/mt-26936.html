TCH Seizure Requests in TCH Handovers (Traffic Channel)

Measurement Counters

ID Counter Description
1278087434 CELL.KPI.TCH.HO.REQ.TRAF K3010B:TCH Seizure Requests in TCH Handovers (Traffic Channel)

Description

In an intra-BSC incoming handover procedure, the MS requests the BSS to reallocate a TCH. In an inter-BSC incoming handover procedure, the MSC requests the BSS to reallocate a TCH. This counter provides the total number of TCH seizure requests in the previous two procedures. This counter is measured in the target cell.

Measurement point

If the BSC decides to initiate an internal intra-cell or inter-cell handover after receiving an MR from an MS, it measures this counter in the target cell, as shown by point D in Figure 1.

Figure 1 Internal inter-cell handover procedure (excluding directed retry)

If the BSC receives an ASSIGNMENT REQUEST message from the MSC and finds that no TCH is available in the local cell, it attempts to trigger a directed retry to a neighboring cell and measures this counter in the target cell, as shown by point D in Figure 2.

Figure 2 Internal inter-cell handover procedure (directed retry)

In the incoming external handover procedure, the BSC that controls the target cell measures the counter in the target cell when it receives a HANDOVER REQUEST from the MSC, as shown by point D in Figure 3.

Figure 3 External handover procedure (excluding directed retry)

If the source BSC (BSC1) receives an ASSIGNMENT REQUEST message from the MSC and finds that no TCH is available in the local cell, it attempts to trigger a directed retry to a cell controlled by another BSC (BSC2). Then, BSC1 sends a HANDOVER REQUIRED message to the MSC, and the MSC sends a HANDOVER REQUEST message to BSC2. BSC2 measures this counter in the target cell when it receives the HANDOVER REQUEST message, as shown by point D in Figure 4.

Figure 4 External handover procedure (directed retry)
