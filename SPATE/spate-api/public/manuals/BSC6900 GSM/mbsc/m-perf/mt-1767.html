Call Drops due to Connection Failures in Stable BSC local switching State

Measurement Counters

ID Counter Description
1278089458 CELL.CALL.DROPS.CONN.FAIL.BSC.LOOP M3701:Call Drops due to Connection Failures in Stable BSC local switching State

Description

After seizing a channel, an MS is in the BSC local switch state. The BTS sends the BSC a CONN FAIL IND message due to the faulty radio links or the faulty hardware, indicating that call drops occur on the Um interface because of the failure in connecting the radio links. This counter provides the number of call drops when the cause value carried in the CONN FAIL IND message is radio link failure (0x01), handover access failure (0x02), om intervention (0x07), or radio resource not available (0x21).

Measurement point

If the BSC receives a CONN FAIL IND message, the BSC regards it as a call drop. The release procedure is started. If the MS is detected in the BSC local switch state, this counter is measured.
