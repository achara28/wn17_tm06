Number of Failed Uplink GPRS TBF Establishments due to MS No Response (Overlaid Subcell)

Measurement Counters

ID Counter Description
1282432198 UP.GPRS.TBF.ESTB.FAIL.MS.NO.RESP.OVERLAY A9041:Number of Failed Uplink GPRS TBF Establishments due to MS No Response (Overlaid Subcell)

Description

This counter provides the number of failed uplink GPRS TBF establishments due to no response from the MS in the overlaid subcell in a measurement period. If the value of this counter is high, the radio network quality of the overlaid subcell is bad.

Measurement point

The uplink GPRS TBF establishment failures due to no response from the MS involve the following conditions:

Failed uplink GPRS TBF establishment using one phase access due to MS no response

After sending an IMMEDIATE ASSIGNMENT message during the procedure of one phase access establishment, the BSC starts to allocate a valid USF for the MS to schedule uplink blocks. If the BSC receives a valid uplink block in the blocks reserved for the MS, the counter N3101 is reset. Otherwise, N3101 is incremented by one, and the BSC reschedules uplink blocks for the MS by sending the Packet Downlink Dummy Control Block message. If N3101 overflows, the BSC releases the uplink GPRS TBF.

Figure 1 shows the procedure in which the uplink GPRS TBF establishment fails because the MS does not respond to the IMMEDIATE ASSIGNMEN message from the BSC. This counter is incremented by one when N3101 overflows (see measurement point A).

Figure 1 Failed uplink GPRS TBF establishment using one phase access due to MS no response

Failed uplink GPRS TBF establishment using single block access due to MS no response

After sending a PACKET UPLINK ASSIGNMENT message during the procedure of single block access establishment, the BSC starts to allocate a valid USF for the MS to schedule uplink blocks. If the BSC receives a valid uplink block in the blocks reserved for the MS, the counter N3101 is reset. Otherwise, N3101 is incremented by one, and the BSC reschedules uplink blocks for the MS by sending the Packet Downlink Dummy Control Block message. If N3101 overflows, the BSC releases the uplink GPRS TBF.

Figure 2 shows the procedure in which the uplink GPRS TBF establishment fails because the MS does not respond to the PACKET UPLINK ASSIGNMENT message from the BSC. This counter is incremented by one when N3101 overflows (see measurement point A).

Figure 2 Failed uplink GPRS TBF establishment using single block access due to MS no response

Uplink GPRS TBF establishment failure on the PACCH due to no MS response

After sending a PACKET UPLINK ASSIGNMENT message on the PACCH, the BSC starts to allocate a valid USF for the MS to schedule uplink blocks. If the BSC receives a valid uplink block in the blocks reserved for the MS, the counter N3101 is reset. Otherwise, N3101 is incremented by one, and the BSC reschedules uplink blocks for the MS by sending the Packet Downlink Dummy Control Block or Downlink Data Block message or downlink control blocks. If N3101 overflows, the BSC releases the uplink GPRS TBF.

Figure 3 shows the procedure in which the uplink GPRS TBF establishment fails because the MS does not respond to the PACKET UPLINK ASSIGNMENT message from the BSC. This counter is incremented by one when N3101 overflows (see measurement point A).
