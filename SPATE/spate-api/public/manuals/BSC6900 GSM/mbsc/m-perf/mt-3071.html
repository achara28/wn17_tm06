Description

This measurement provides the total number of uplink RLC data blocks using CS1, CS2, CS3, and CS4 in a cell in a granularity period. The more data blocks using robust coding scheme are, the higher transmission rate is. The robustness of the coding scheme is related to the quality of the Um interface. Higher quality of the Um interface indicates a large number of data blocks using robust coding scheme.

The following table describes the meaning of each counter.

Counter Description
UP.CS1.RLC.BLK Total Number of Uplink RLC Data Blocks Using CS1
UP.CS2.RLC.BLK Total Number of Uplink RLC Data Blocks Using CS2
UP.CS3.RLC.BLK Total Number of Uplink RLC Data Blocks Using CS3
UP.CS4.RLC.BLK Total Number of Uplink RLC Data Blocks Using CS4

Measurement point

The total number of uplink RLC data blocks using CS1 coding scheme that is sent from the MS is calculated in the following methods:

When the BSC receives an RLC data block using CS1 coding scheme, and the data block in the V(N) on the BSC side is in NACKED state , you can infer that the data block has been sent several times. In this case, the counter is incremented by two.

When the BSC receives an RLC data block using CS1 coding scheme, and the data block in the V(N) on the BSC side is in UNRECEIVED state, you can infer that the data block has been sent once only. In this case, the counter is incremented by one.

When the BSC receives an RLC data block using CS1 coding scheme, and the data block in V(N) on the BSC side is in RECEIVED state, you can infer that the data block is received. In this case, the counter is incremented by one.

The measurement point of Total Number of Uplink RLC Data Blocks Using CS2, CS3, or CS4 coding scheme is the same as that of Total Number of Uplink RLC Data Blocks Using CS1.
