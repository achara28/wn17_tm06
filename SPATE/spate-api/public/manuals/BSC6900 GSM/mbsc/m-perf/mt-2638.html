Number of Network-Controlled Cell Reselection Attempts

Measurement Counters

ID Counter Description
1279172426 NC.CELL.RESELECT.ATMPT H9010:Number of Network-Controlled Cell Reselection Attempts

Description

This measurement provides the number of network-controlled cell reselection attempts in a granularity period. The value of this counter is related to cell reselection policies such as the radio access technology (RAT) used over the Um interface, downlink transmission quality, cell load, and neighboring cell level.

Measurement point

The NC2 cell reselection algorithm consists of service-based cell reselection, quality-based cell reselection, load-based cell reselection, coverage-based cell reselection, and forced cell reselection. Service-based cell reselection enables an MS to reselect to a 3G or LTE cell if the type of the ongoing service is not supported in the GSM cell. Quality-based cell reselection can decrease the number of service interruptions if the radio environment degrades sharply. Load-based cell reselection allows a cell to share the traffic load of another cell and therefore prevents some counters from deteriorating. Coverage-based cell reselection enables the network to provide high quality services to MS while maintaining the MS mobility. Forced cell reselection enables a dual-mode MS to reselect to a 3G cell to process PS services as best as possible and therefore improves the 3G network efficiency and user experience and reduces the GSM cell load. The BSC determines whether to trigger cell reselection based on the information in the MRs from the MS, value of the Service UTRAN CCO IE sent from the SGSN, and the mode priority and conditions for the serving cell. This counter is incremented by one when service-based cell reselection, quality-based cell reselection, load-based cell reselection, coverage-based cell reselection, or forced cell reselection is triggered.
