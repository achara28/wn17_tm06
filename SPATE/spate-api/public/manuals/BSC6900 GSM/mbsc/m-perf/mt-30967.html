Percentage of Duration When Delay Jitter of RTP Packets Exceeds the Threshold to Total Activation Duration

Measurement Counters

ID Counter Description
73441287 VS.IPPOOL.RTP.JitterOverrun.TimeRatio P141016:Percentage of Duration When Delay Jitter of RTP Packets Exceeds the Threshold to Total Activation Duration

Description

This counter provides the percentage of the duration when the delay jitter of RTP packets received over the Iu/A interface between a certain pair of IP addresses exceeds the threshold to the total activation duration in a measurement period.

Measurement point

This counter is calculated by using the following formula in scenarios where Iu/A over IP is applied and RTCP is supported: Percentage of duration when delay jitter for receiving RTP packets exceeds threshold = Duration when delay jitter for receiving RTP packets exceeds threshold/Total activation duration where, "Total activation duration" is the sum of activation periods of all services processed between the IP address pair. "Duration when delay jitter for receiving RTP packets exceeds threshold" is the total duration when the delay jitters for receiving RTP packets between the IP address pair exceeds the threshold. The delay for receiving an RTP packet refers to the duration from the time when the RTP packet is sent by the peer end to the time when the RTP packet is received at the local end.

Formula

None

per mill
