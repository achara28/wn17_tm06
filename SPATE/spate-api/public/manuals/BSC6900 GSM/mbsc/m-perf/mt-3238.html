Number of Dynamic PDCH Requests Without Application Attempts Because No Abis Timeslot is Available for Use

Measurement Counters

ID Counter Description
1279270463 NEED.DYN.PDCH.FAIL.WITHOUT.USEABLE.ABIS R9347:Number of Dynamic PDCH Requests Without Application Attempts Because No Abis Timeslot is Available for Use

Description

This counter provides the number of failed dynamic PDCH requests in a cell in a granularity period because no Abis timeslot is available for use.

If multiple requests fail or the rate of successful dynamic PDCH requests is lower than 80%, you can infer that the traffic of the BSC is high or the channel state of the BSC is abnormal. One of the following methods can be implemented: Perform the capacity expansion to increase Abis timeslots. Set the uplink and downlink PDCH multiplexing thresholds to higher values. Set Maximum Rate Threshold of PDCHs to a lower value.

Measurement point

In dynamic channel request process during PS services, if no Abis timeslot is available for use, this counter is incremented by one.
