Description

This counter provides the number of failed SRVCC handovers in the target cell based on the type of the target channel and the cause of the failure. Together with SRVCC handover requests and SRVCC handover responses, these counters indicate the performance of the SRVCC handover.

This counter is measured when the SRVCC handover is terminated. The channel involved in the measurement can be the SDCCH or TCH.

  • SR1004:CELL.SRVCC.INCELL.HO.FAIL.SD
  • SR1005:CELL.SRVCC.INCELL.HO.FAIL.TCH
The following counters provide the number of failed SRVCC handovers due to specific causes. These counters are measured upon the termination of the SRVCC handover.

(1) The following counter provides the number of failed SRVCC handovers because the BSC fails to decode an HO REQ message with invalid contents from the MSC.

  • SR1006:CELL.SRVCC.INCELL.HO.FAIL.INVALID.CONTENTS
(2) The following counters provide the numbers of failed SRVCC handovers based on the causes related to resource allocations.

The following counters provide the numbers of failed SRVCC handovers because the SDCCH or TCH is unavailable in the target cell. The specific counter is measured when the channel allocation fails.

  • SR1007:CELL.SRVCC.INCELL.HO.FAIL.CONG.SD
  • SR1008:CELL.SRVCC.INCELL.HO.FAIL.CONG.TCH
The following counter provides the number of failed SRVCC handovers because no proper speech version is available based on the speech versions carried in the HO REQ message from the MSC, the speech versions supported by the target cell, and the speech versions permitted by the A interface phase tag. It is measured when no speech version is available.
  • ]SR1016:CELL.SRVCC.INCELL.HO.FAIL.NO.SPEECH.VER
The following counter provides the number of failed SRVCC handovers because the BSC fails to obtain the requested circuit resource when establishing the circuit connection.
  • SR1011:CELL.SRVCC.INCELL.HO.FAIL.APPLY.TRSL.FAIL
The following counter provides the number of failed SRVCC handovers because the specified CIC on the A interface is unavailable, after the BSC receives the HO REQ message from the MSC.
  • SR1009:CELL.SRVCC.INCELL.HO.FAIL.NO.CIC
The following counter provides the number of failed SRVCC handovers because the specified CIC on the A interface has already been allocated to another MS, after the BSC receives the HO REQ message from the MSC.
  • SR1010:CELL.SRVCC.INCELL.HO.FAIL.CIC.ALLOC
(3) The following counters provide the numbers of failed SRVCC handovers because timer T3121 expires. Timer T3121 is started after the BSC allocates and activates a channel in the target cell and sends an HO REQ ACK message to the MSC. These counters are measured in the target cell based on the channel type and service type of the target channel (SDCCH, TCH as signaling channel, or TCH as traffic channel).
  • SR1013:CELL.SRVCC.INCELL.HO.FAIL.EXP.SD
  • SR1014:CELL.SRVCC.INCELL.HO.FAIL.EXP.TCH.SIG
  • SR1015:CELL.SRVCC.INCELL.HO.FAIL.EXP.TCH.TRAF.CH
(4) The following counter provides the number of failed SRVCC handovers because the BSC releases the call after receiving the CLEAR CMD message from the MSC.
  • SR1012:CELL.SRVCC.INCELL.HO.FAIL.MSC.CLR.CMD
The main causes of an SRVCC handover failure include:

(1) Causes related to message decoding

  • The BSC fails to decode the HO REQ message.
2) Causes related to resource allocation
  • No traffic channel is available.
  • No speech version is available.
  • The BSC fails to obtain the terrestrial resources.
  • No CIC is available.
  • The CIC is already allocated.
(3) Causes related to access on the Um interface
  • The timer for the BSC to wait for an HO CMP message expires.
(4) Other causes
  • The BSC releases the call after receiving the CLR CMD message from the MSC.

Measurement point

The following counters are measured when an SRVCC handover fails after the BSC receives an HO REQ message from the MSC, respectively because the message is invalid, no suitable channel is available in the target cell, or no proper speech version is available:

  • SR1006:CELL.SRVCC.INCELL.HO.FAIL.INVALID.CONTENTS
  • SR1007:CELL.SRVCC.INCELL.HO.FAIL.CONG.SD
  • SR1008:CELL.SRVCC.INCELL.HO.FAIL.CONG.TCH
  • SR1016:CELL.SRVCC.INCELL.HO.FAIL.NO.SPEECH.VER
The following counters are measured when an SRVCC handover fails because the specified CIC is unavailable or has already been allocated to another MS, or the BSC fails to establish the terrestrial link:
  • SR1009:CELL.SRVCC.INCELL.HO.FAIL.NO.CIC
  • SR1010:CELL.SRVCC.INCELL.HO.FAIL.CIC.ALLOC
  • SR1011:CELL.SRVCC.INCELL.HO.FAIL.APPLY.TRSL.FAIL
The following counters are measured when an SRVCC handover fails because timer T3121 expires after the BSC sends an HO REQ ACK message to the MSC:
  • SR1013:CELL.SRVCC.INCELL.HO.FAIL.EXP.SD
  • SR1014:CELL.SRVCC.INCELL.HO.FAIL.EXP.TCH.SIG
  • SR1015:CELL.SRVCC.INCELL.HO.FAIL.EXP.TCH.TRAF.CH
The following counter is measured when an SRVCC handover fails because the BSC releases the call after receiving the CLEAR CMD message from the MSC:
  • SR1012:CELL.SRVCC.INCELL.HO.FAIL.MSC.CLR.CMD
The following counters are measured when an SRVCC handover fails:
  • SR1004:CELL.SRVCC.INCELL.HO.FAIL.SD
  • SR1005:CELL.SRVCC.INCELL.HO.FAIL.TCH
