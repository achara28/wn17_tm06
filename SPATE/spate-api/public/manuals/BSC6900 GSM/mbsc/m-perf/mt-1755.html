Number of Attempts to Return to Normal Call from BTS local switching

Measurement Counters

ID Counter Description
1278089439 CELL.LOCALSWITCH.RESTORE.TRY.BTS.LOOP R3841:Number of Attempts to Return to Normal Call from BTS local switching

Description

After the BTS local switching starts, the BTS local switching recovery is initiated in the following cases:

1. After the BTS local switching starts, a call processing module at one end needs to perform data services. The BSC receives a data service assignment request message from the MSC. The local switching, however, does not support data services. Therefore, the call should be returned to a normal one before the data services can be performed. Within the BSC, the call processing module that needs to perform data services initiates a local switching recovery request. Then, the local switching recovery procedure is started.

2. When the calling and called MSs are in the stable BTS local switching state, one MS performs an outgoing BSC handover. The cross-BSC local switching is not supported. Therefore, after receiving a handover command from the MSC, the BSC notifies the call processing module at the peer end that the call should be returned to a normal one to ensure the speech quality after the outgoing BSC handover is successful. If the BSC receives a handover failure message from the MS, the call processing module initiating the handover is also restored to the normal call state. Within the BSC, the call processing module initiating the outgoing BSC handover initiates a local switching recovery request. Then, the local switching recovery procedure is started.

3. When the calling and called MSs are in the stable BTS local switching state, one of them initiates an intra-BSC handover. After receiving the handover detection indication message, the BSC returns the local switching call to a normal one if the handover fails. Within the BSC, the call processing module initiating the handover initiates a local switching recovery indication. Then, the local switching recovery procedure is started.

4. When the calling and called MSs are in the stable BTS local switching state, a local switching request is initiated again. The local switching recovery is performed if the local switching acknowledgement message due to the following causes is received: Timer Expired, Failures on the BTS Side, and Other Causes.

This counter provides the number of attempts to return to normal call from BTS local switching.

During the process where the local switching is restored to a normal call, the transmission channel between the Ater interface and the Abis interface should be reconnected and TC resources should be requested. The speech signals reach the MSC through the BSC, and are switched to the call processing module at the peer end. If the reconnection fails, the cause value in the local switching acknowledgement message is Terrestrial Resource Request Failure. Then, the BSC regards it as a call drop and then releases the call.
