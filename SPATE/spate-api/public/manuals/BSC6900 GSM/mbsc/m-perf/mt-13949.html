Description

This counter provides the total number of valid downlink RLC data blocks using CS1-CS4 in a subsite in a measurement period.

The following table describes the meaning of each counter.

Counter Description
DOWN.CS1.RLC.BLK.VALID.TOTAL.LOCGRP Total Number of Valid Downlink RLC Data Blocks Using CS1 in a Subsite
DOWN.CS2.RLC.BLK.VALID.TOTAL.LOCGRP Total Number of Valid Downlink RLC Data Blocks Using CS2 in a Subsite
DOWN.CS3.RLC.BLK.VALID.TOTAL.LOCGRP Total Number of Valid Downlink RLC Data Blocks Using CS3 in a Subsite
DOWN.CS4.RLC.BLK.VALID.TOTAL.LOCGRP Total Number of Valid Downlink RLC Data Blocks Using CS4 in a Subsite

Measurement point

Use the Total Number of Valid Downlink RLC Data Blocks Using CS1 in a Subsite counter as an example.

The measurement is triggered at point A in figure 1 (the process that the BSC sends a downlink GPRS CS1 RLC data block and receives a PACKET DOWNLINK ACK/NACK message). After receiving the message from the MS, the BSC determines whether the MS receives the downlink GPRS CS1 RLC data block based on the bitmap in the PACKET DOWNLINK ACK/NACK message. This counter is incremented by one the first time the BSC confirms that the MS receives the data block.
