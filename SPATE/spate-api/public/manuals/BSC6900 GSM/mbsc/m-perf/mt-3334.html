Call Drops on Radio Interface (SDCCH )(Call Type) in One Location Group

Measurement Counters

ID Counter Description
1282412040 CELL.LOCGRP.SD.UM.CALL.DROPS.CALL TCM30D:Call Drops on Radio Interface (SDCCH )(Call Type) in One Location Group

Description

After the MS seizes a channel, if a call is released because of Abis terrestrial link failure, no MRs from MS for a long time, equipment failure, or forced handover failure, this counter provides the number of call drops on the SDCCHs in the location group on the basis of the call type (MOC, MTC, call re-establishment, emergency call, or non-call).

Measurement point

When an SDCCH is seized during a call, and the call type is MOC, MTC, emergency call, or call re-establishment, this counter is measured in the following situations:
  • The BSC receives an ERROR INDICATION message from the BTS because of the abnormal connection on the radio link layer. For details, refer to GSM Rec. 08.58.
  • The BSC receives a CONNECTION FAILURE INDICATION message from the BTS because of link fault or hardware fault on the radio link layer. For details, refer to GSM Rec. 08.58.
  • The BSC receives a REL IND message from the BTS. Before this, the BTS has received a DISC frame from the MS in the multi-frame link set-up state. For details, refer to GSM Rec. 08.58.
  • The timer set to wait for a HO CMP message expires during internal intra-cell handover.
  • The timer set to wait for a HO CMP message expires during outgoing internal inter-cell handover.
  • The timer set to wait for a HO CMP message expires during outgoing external inter-cell handover.
  • The timer set to wait for a HO CMP message expires during incoming external inter-cell handover.
