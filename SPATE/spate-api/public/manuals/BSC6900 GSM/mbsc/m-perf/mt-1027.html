Description

If the incoming external inter-cell handover fails during the handover procedure, the specific counter is measured in the target cell under control of the target BSC, based on the type of the target channel and the cause of the failure. Together with Incoming External Inter-Cell Handover Requests and Incoming External Inter-Cell Handover Responses, this counter indicates the performance of the incoming external inter-cell handover.

The following counters provide the numbers of failed incoming internal inter-cell handovers when the handover occurs between the frequency bands. The target channel for handover can be SDCCH or TCH. They are measured when the incoming internal inter-cell handover is terminated.

  • H3420W:CELL.INTERBSC.INCELL.HO.FAIL.SD.900.900
  • H3420X:CELL.INTERBSC.INCELL.HO.FAIL.SD.1800.1800
  • H3420Y:CELL.INTERBSC.INCELL.HO.FAIL.SD.900.1800
  • H3420Z:CELL.INTERBSC.INCELL.HO.FAIL.SD.1800.900
  • H3429W:CELL.INTERBSC.INCELL.HO.FAIL.TCH.900.900
  • H3429X:CELL.INTERBSC.INCELL.HO.FAIL.TCH.1800.1800
  • H3429Y:CELL.INTERBSC.INCELL.HO.FAIL.TCH.900.1800
  • H3429Z:CELL.INTERBSC.INCELL.HO.FAIL.TCH.1800.900
The following counters provide the numbers of failed incoming external inter-cell handovers due to several common causes. The specific counter is measured when the incoming external inter-cell handover is terminated.

1) The following counter provides the number of failed incoming external inter-cell handovers because the BSC fails to decode an HO REQ message from the MSC:

  • H342I:CELL.INTERBSC.INCELL.HO.FAIL.CONTENTS.INVALID
2) The following counters provide the numbers of failed incoming external inter-cell handover based on the causes related to resource allocations.

The following counters provide the number of failed incoming external inter-cell handovers because the SDCCH or TCH is unavailable in the target cell. It is measured when the channel allocation fails.

  • H3420A:CELL.INTERBSC.INCELL.HO.FAIL.CONG.SD
  • H3429A:CELL.INTERBSC.INCELL.HO.FAIL.CONG.TCH
The following counter provides the number of failed incoming external inter-cell handovers because no Abis circuit resource is available in the target cell when the Abis dynamic allocation is enabled.
  • H342L:CELL.INTERBSC.INCELL.HO.FAIL.NO.IDLE.ABIS
The following counter provides the number of failed incoming external inter-cell handovers because no proper speech version is available based on the speech versions carried in the HO REQ message from the MSC, the speech versions supported by the target cell, and the speech versions permitted by the A interface phase tag. It is measured when no speech version is available.
  • H342J:CELL.INTERBSC.INCELL.HO.FAIL.NO.SPEECH.VER
The following counter provides the number of failed incoming external inter-cell handovers because the BSC fails to obtain the circuit resource when establishing the terrestrial connection.
  • H342B:CELL.INTERBSC.INCELL.HO.FAIL.APPLY.TRSL.FAIL
The following counter provides the number of failed incoming external inter-cell handovers because the CIC on the A interface specified in the HO REQ message is unavailable, after the BSC receives the HO REQ from the MSC.
  • H342E:CELL.INTERBSC.INCELL.HO.FAIL.NO.CIC
The following counter provides the number of failed incoming external inter-cell handovers because the CIC on the A interface specified in the HO REQ message has already been allocated to another MS, after the BSC receives the HO REQ from the MSC.
  • H342F:CELL.INTERBSC.INCELL.HO.FAIL.CIC.AVAIL
3) The following counters provide the numbers of failed incoming external inter-cell handovers because the MS fails to access the new channel on the Um interface.

During incoming external inter-cell handover procedure, after successfully allocating and activating the channel in the target cell, the BSC sends an HO REQ ACK message to the MSC and starts the timer to wait for the HO CMP message. If no HO COMP message is received before the timer expires, the BSC releases the call. The specific counter provides the number of failed incoming external inter-cell handovers, based on the channel type and service type of the target channel (SDCCH, TCH as signaling channel, or TCH as traffic channel) in this scenario.

  • H3420C:CELL.INTERBSC.INCELL.HO.FAIL.EXP.SD
  • H3429Cb:CELL.INTERBSC.INCELL.HO.FAIL.EXP.TCH.SIG
  • H3429Ca:CELL.INTERBSC.INCELL.HO.FAIL.EXP.TCH.TRAF
(4) The following counter provides the number of failed incoming external inter-cell handovers because the BSC releases the call after receiving the CLEAR CMD message from the MSC.

If the incoming external inter-cell handover fails because the BSC releases the call after receiving the CLEAR CMD message from the MSC, the following counter is measured:

  • H342H:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD
If the incoming external inter-cell handover fails because the BSC releases the call after receiving the CLEAR CMD message from the MSC, the following counter are measured based on the cause values of the CLEAR CMD message:
  • H342HA:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.UM.MSG.FAIL
  • H342HB:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.UM.FAIL
  • H342HC:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.OM
  • H342HD:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.EQP.FAIL
  • H342HE:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.PREEMPTION
  • H342HF:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.INVAILD.MSG
  • H342HG:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.PROTOCOL.ERR
  • H342HH:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.OTHER.FAIL
The main causes for a failed incoming external inter-cell handover include:
1. Causes related to message decoding
  • The BSC fails to decode the HO REQ message
2. Causes related to resource allocation
  • No traffic channel is available
  • No Abis resource is available
  • No speech version is available
  • The BSC fails to obtain the terrestrial resources
  • No CIC is available
  • The CIC is already allocated
3. Causes related to access on the Um interface
  • MCS releases the call because of failed access on the radio interface
  • The timer for the BSC to wait for an HO CMP message expires
4. Other causes
  • The BSC releases the call after receiving the CLR CMD from the MSC
The number of failed incoming external inter-cell handovers due to rare causes, such as equipment failures and internal checks, is not measured. You can calculate the number of such failed handovers by subtracting the number of failed handovers due to the preceding common causes from the value of Failed Incoming External Inter-Cell Handovers.

Measurement points Description

C The timer waiting for the handover completion expires.

As shown in Figure 1, if the incoming external inter-cell handover fails, the specific one of the following counters is measured at A, respectively because the message has an invalid content, no suitable channel (SDCCH/TCH) is available in the target cell, no available speech version, or no Abis interface resource is available:

  • H342I:CELL.INTERBSC.INCELL.HO.FAIL.CONTENTS.INVALID

  • H3429A:CELL.INTERBSC.INCELL.HO.FAIL.CONG.TCH

  • H342J:CELL.INTERBSC.INCELL.HO.FAIL.NO.SPEECH.VER
As shown in Figure 1, the specific counter is measured at B if the incoming external inter-cell handover fails because the BSC fails to apply for the terrestrial link or no circuit resources is available:
  • H342B:CELL.INTERBSC.INCELL.HO.FAIL.APPLY.TRSL.FAIL
  • H342E:CELL.INTERBSC.INCELL.HO.FAIL.NO.CIC
  • H342F:CELL.INTERBSC.INCELL.HO.FAIL.CIC.AVAIL
As shown in Figure 1, during the incoming external inter-cell handover procedure, after the BSC sends the HO CMD to the MS, the specific counter is measured at C if the timer for the HO CMP message expires:
  • H3420C:CELL.INTERBSC.INCELL.HO.FAIL.EXP.SD
  • H3429Cb:CELL.INTERBSC.INCELL.HO.FAIL.EXP.TCH.SIG
  • H3429Ca:CELL.INTERBSC.INCELL.HO.FAIL.EXP.TCH.TRAF
As shown in Figure 1, during the incoming external inter-cell handover procedure, if the BSC releases the call after receiving the CLEAR CMD from the MSC, the following counter is measured at D:
  • H342H:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD
The counter is measured when the cause value carried in the CLEAR CMD message is Radio Interface Message Failure.
  • H342HA:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.UM.MSG.FAIL
The counter is measured when the cause value carried in the CLEAR CMD message is Radio Interface Failure.
  • H342HB:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.UM.FAIL
The counter is measured when the cause value carried in the CLEAR CMD message is OM Intervention.
  • H342HC:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.OM
The counter is measured when the cause value carried in the CLEAR CMD message is Equipment Failure.
  • H342HD:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.EQP.FAIL
The counter is measured when the cause value carried in the CLEAR CMD message is Preemption.
  • H342HE:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.PREEMPTION
The counter is measured when the cause value carried in the CLEAR CMD message is Invalid Message.
  • H342HF:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.INVAILD.MSG
The counter is measured when the cause value carried in the CLEAR CMD message is Protocol Error between BSS and MSC.
  • H342HG:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.PROTOCOL.ERR
The counter is measured when the cause value carried in the CLEAR CMD message is Other Causes.
  • H342HH:CELL.INTERBSC.INCELL.HO.FAIL.MSC.CLR.CMD.OTHER.FAIL
