Number of HSDPA MIMO+64QAM RABs Normal Released for cell

Measurement Counters

ID Counter Description
73403827 VS.HSDPA.RAB.NormRel.MIMO64QAM Number of HSDPA MIMO+64QAM RABs Normal Released for cell

Description

This measurement item provides the number of HSDPA MIMO+64QAM RABs normal released for cell.For DC-HSDPA users, this counter is measured only in the cell with the primary carrier.

Measurement point

The measurement is triggered at point A as shown in the figure. When the RNC receives an IU RELEASE COMMAND message, it releases the related RABs and sends an IU RELEASE COMPLETE message to the CN. If the IU RELEASE COMMAND message indicate the reason is "User Inactivity","Normal Release","Successful Relocation" or "Network Optimization" , or the RNC initiates the release procedure by sending an IU RELEASE REQUEST message or a RAB RELEASE REQUEST message with the causes of "User Inactivity","Network Optimization" or "UE Generated Signalling Connection Release",the RNC measures this item in the HSDPA cell if it is a HSDPA service using MIMO+64QAM.

Figure 1 Iu release procedure

The measurement is triggered at point B as shown in the figure.When the RNC receives a RAB ASSIGNMENT REQUEST message indicating that some RABs need to be released, the RNC releases the related RABs and sends a RAB ASSIGNMENT RESPONSE message to the CN. If the IU RELEASE COMMAND message indicate the reason is "User Inactivity","Normal Release","Successful Relocation" or "Network Optimization" , or the RNC initiates the release procedure by sending an IU RELEASE REQUEST message or a RAB RELEASE REQUEST message with the causes of "User Inactivity","Network Optimization" or "UE Generated Signalling Connection Release" ,the RNC measures this item in the HSDPA cell if it is a HSDPA service using MIMO+64QAM.
