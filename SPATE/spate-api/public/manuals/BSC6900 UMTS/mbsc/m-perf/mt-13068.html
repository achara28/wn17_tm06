Description

These counters provide the number of retransmitted RRC connection setup requests in the cell after an RRC connection request is rejected due to resource insufficiency. These counters indicate whether the resources are congested. A high counter value indicates that a large number of users have accessed the cell or the resources of the RNC or NodeB are insufficient, and the RRC connection setup fails.

Measurement point

The measurement is triggered at point B, C, D, and so on as shown in the figure. If an RRC Connection Request message sent by the UE is rejected because of resource insufficiency, the UE sends the RRC Connection Request message again. In this case, a counter is measured according to the request cause as shown in point B. If the request is rejected again, the UE sends another RRC Connection Request message, and a counter is measured again according to the request cause as shown in point C. That is, each time an RRC Connection Request message is retransmitted within (RRCWaitTime x N300 + 2) seconds after the last RRC Connection Request message is sent by the UE, a counter is measured.
