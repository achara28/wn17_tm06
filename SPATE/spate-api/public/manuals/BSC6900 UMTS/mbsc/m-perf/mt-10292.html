Description

The measurement items provide the number of failed CS RABs due to radio resource congestion in the best cell.If congestion occurs on both the uplink and downlink, only the related downlink counter is measured. For example, if a CS RAB connection setup fails due to the congestion on both the uplink and downlink, only the counter VS.RAB.FailEstabCS.DLPower.Cong is measured.

Measurement point

The measurement is triggered at point B as shown in the figure, If the RNC detects a request to set up RABs in the RAB ASSIGNMENT REQUEST message from the CN, it requests related radio resources. If the request for radio resources fails, the RNC regards the setup of RABs as a failure and sends a RAB ASSIGNMENT RESPONSE message to the CN(With failure cause of "No Radio Resources Available in Target cell","Requested Maximum Bit Rate not Available","Requested Maximum Bit Rate for UL not Available","Requested Guaranteed Bit Rate not Available","Requested Guaranteed Bit Rate for DL not Available" or "Requested Guaranteed Bit Rate for UL not Available"). In this case, the RNC measures each item in the best cell according to different causes of radio resource congestion.
