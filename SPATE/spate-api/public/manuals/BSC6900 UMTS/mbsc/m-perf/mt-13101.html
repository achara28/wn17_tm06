Description

These measurement items provides the number of times that CS or PS Service Releases Caused by the second RRC Connection Request from the same UE for Cell.

VS.Reserve.Counter1: provides the number of times that CS Service Releases Caused by the second RRC Connection Request from the same UE for Cell.

VS.Reserve.Counter2: provides the number of times that PS Service Releases Caused by the second RRC Connection Request from the same UE for Cell.

VS.Reserve.Counter3: provides the number of times that PS Service Releases Caused by the second RRC Connection Request from the same UE in a Cell when PS services are carried over common channels.

Note: Second RRC Connection Request: The UE sends a second RRC CONNECTION REQUEST message to the RNC before releasing the previously assigned RAB.

Measurement point

As shown in point A of the figure, after receiving an RRC CONNECTION REQUEST message from a UE, the RNC determines that the message is sent by the same UE with RAB using a COMMAND ID message sent by the CN, and then the RNC sends an IU RELEASE REQUEST message with the cause value of "UE Generated Signaling Connection Release". Upon reception of the IU RELEASE REQUEST message, the CN sends an IU RELEASE COMMAND message to the RNC. After receiving this message, the RNC releases the corresponding RAB and sends an IU RELEASE COMPLETE message to the CN. In this case, the value of counter VS.Reserve.Counter1 or VS.Reserve.Counter2 is measured based on the service type in the best cell that the UE camps on. When UE's channel transition (including D2P,D2F) succeeds, or UE is in the CELL_FACH state, CELL_PCH state, the value of counter VS.Reserve.Counter3 is measured in the best cell that the UE camps on.
