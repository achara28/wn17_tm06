Description

These counters provide the number of bytes received or transmitted at the data link layer for the trunk group in a specified measurement period.

They indicate the traffic volume at the data link layer for the trunk group in the specified measurement period. The number of bytes measured includes the MAC header, payload, and Frame Check Sequence (FCS).

Normally, if the networking scenario remains unchanged, the traffic carried on the trunk group is constant at the same period every day. Also, values of these counters are constant. If the number of received and transmitted bytes decreases drastically and the number of bytes incorrectly received and transmitted increases sharply, you can infer that a transmission fault occurs.

Related counters: Receive Rate and Transmit Rate of Trunk Group, BER of Data Transmitted and Received by Trunk group

Measurement point

Samples are taken every five seconds in the specified measurement period. The system obtains the number of bytes received and transmitted on the port once every five seconds, and calculates the total number of bytes that are received, transmitted, or discarded in one measurement period.
