Description

These counters provide the receive rate and transmit rate of the trunk group at the data link layer in a specified measurement period.

They indicate the traffic volume at the data link layer for the trunk group in the specified measurement period. The number of bytes measured includes the number of bytes in the header of the MAC frame and the payload.

The values of these counters are displayed as integers with decimals rounded down. For example, if the value of one of these counters is 9.9 kbit/s, the value is displayed as 9 kbit/s.

Normally, if the networking scenario remains unchanged, the traffic carried on the trunk group is constant at the same period every day. Also, values of these counters are constant. If both the receive rate and transmit rate of the trunk group decrease greatly, you can infer that a transmission fault occurs.

Related counters: Number of Bytes That Are Transmitted and Received by Trunk Group, BER of Data Transmitted and Received by Trunk Group

Measurement point

Samples are taken every five seconds in the specified measurement period. The system obtains the number of bytes that are transmitted by the trunk group and the number of bytes that are received by the trunk group every five seconds. The maximum transmit rate can be obtained by dividing the maximum number of transmitted bytes in five seconds by five. The minimum transmit rate can be obtained by dividing the minimum number of transmitted bytes in five seconds by five. The maximum receive rate and the minimum receive rate can also be calculated in this method. The mean transmit rate can be calculated by dividing the total number of transmitted bytes by the measurement period. The mean receive rate can also be calculated in this method.
