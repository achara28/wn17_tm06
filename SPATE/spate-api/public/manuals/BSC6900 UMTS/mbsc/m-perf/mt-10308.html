Description

The measurement items provide the number of failed PS RABs in the best cell due to radio resource congestion for different causes.If congestion occurs on both the uplink and downlink, only the related downlink counter is measured. For example, if a PS RAB connection setup fails due to the congestion on both the uplink and downlink, only the counter VS.RAB.FailEstabPS.DLPower.Cong is measured.

Measurement point

The measurement is triggered at point B as shown in the figure. After the RNC receives a RAB ASSIGNMENT REQUEST message from the CN, if the message contains a request to set up a RAB, the RNC requests for the corresponding radio resources. If the requests fails, the RNC considers the RAB setup as a failure and sends a RAB ASSIGNMENT RESPONSE message to the SGSN(With failure cause of "No Radio Resources Available in Target cell","Requested Maximum Bit Rate not Available","Requested Maximum Bit Rate for UL not Available","Requested Guaranteed Bit Rate not Available","Requested Guaranteed Bit Rate for DL not Available" or "Requested Guaranteed Bit Rate for UL not Available"). In this case, the RNC measures the items in the best cell according to the failure causes.
