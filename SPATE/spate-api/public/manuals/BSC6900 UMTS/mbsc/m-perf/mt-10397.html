Description

This measurement item provides the number of CS RABs abnormally released triggered by different causes in the best cell.

The following table describes the meaning of each counter.

Counter Description
VS.RAB.AbnormRel.CS.IuAAL2 Number of CS RABs Abnormally Released Due to Iu Interface AAL2 Link Failure for Cell
VS.RAB.AbnormRel.CS.RF Number of CS RABs Abnormally Released Due to RF for Cell(The RF failure includes Radio Connection With UE Lost and Failure in the Radio Interface Procedure)
VS.RAB.AbnormRel.CS.OM Number of CS RAB Release Requests Due to OM Intervention for Cell
VS.RAB.AbnormRel.CS.Preempt Number of CS RAB Release Attempts Due to RAB Preemption for Cell
VS.RAB.AbnormRel.CS.UTRANgen Number of CS RAB Abnormal Releases Triggered by RNC Due to UTRAN Cause for Cell
VS.RAB.AbnormRel.CS.OLC Number of CS RAB Release Attempts Due to Congestion for Cell

Measurement point

The measurement is triggered at point A as shown in the figure. The RNC sends to the CN a RAB RELEASE REQUEST message due to exception after the CS RABs have been set up. If the RNC receives a RAB ASSIGNMENT REQUEST message which indicates releasing a RAB with the cause other than "User Inactivity", "Normal Release", "Successful Relocation", "Network Optimization", or "UE Generated Signalling Connection Release"(reference to 3GPP TS 25.413),the RNC measures these items in the best cell that the UE camps on according to different causes.
