Load Balancing of the MSC Pool on the CS Domain

Measurement Counters

ID Counter Description
67192538 VS.IU.LdBalRtCS.IMSI Number of Times that CS Load Balancing is Performed by IMSI Users
67192539 VS.IU.LdBalRtCS.InValidNRI Number of Times that CS Load Balancing is Performed by TMSI Users Due to Invalid NRI
67192540 VS.IU.LdBalRtCS.IMEI Number of Times that CS Load Balancing is Performed by IMEI Users

Description

The measurement counters provide the number of times that the UEs perform load balancing on the CS domain during CN node selection when the RNC processes the initial direct transfer message.

The following table describes the meaning of each counter.

Counter Description
VS.IU.LdBalRtCS.IMSI Number of times that the CS CN is selected for load balancing by using the IMSI route parameter.
VS.IU.LdBalRtCS.InValidNRI Number of times that load balancing is triggered due to invalid NRI route parameter, connection failure to the CN node corresponding to the route parameter, or the flow control of the CN node corresponding to the route parameter.
VS.IU.LdBalRtCS.IMEI Number of times that the CS CN is selected for load balancing by using the IMEI route parameter.

Measurement point

The measurement is triggered when the RNC sends an INITIAL UE MESSAGE message to the CN for load balancing, as shown by point A in the following figure. The corresponding counter is measured on the basis of the UE identification carried in the INITIAL DIRECT TRANSFER message from the UE to the RNC.
