Number of Successful PTT Call Initiation Attempts or Responses During a P2D Procedure for UEs Processing PTT Services

Measurement Counters

ID Counter Description
73423555 VS.PTT.PCH.SingleRAB.CallReq.Succ Number of Successful PTT Call Initiation Attempts or Responses of UEs Using Only the PTT Service in the CELL_PCH or URA_PCH State During the State Transition from CELL_PCH/URA_PCH to CELL_DCH for Cell

Description

This counter measures the number of successful PTT call initiation attempts or responses during a P2D procedure in a cell. This counter is used to analyze PTT service establishment of UEs that send CELL UPDATE messages to initiate or respond to PTT calls.

Measurement point

A UE in the CELL_PCH or URA_PCH state can initiate or respond to a PTT call by sending a CELL UPDATE message during a P2D procedure. The measurement is triggered at point B shown in Figure 1 when the RNC receives a RADIO BEARER RECONFIGURATION COMPLETE message from the UE that initiates or responds to a PTT call during a P2D procedure. A UE sends a CELL UPDATE message to initiate a PTT call when all the following conditions are met: The UE has set up a PTT service. The values of the IEs Cell update cause, Establishment cause, and Traffic volume indicator contained in this message are uplink data transmission, Originating Conversational Call, and TRUE, respectively. This message does not contain the IE CS Call Type. A UE sends a CELL UPDATE message to respond to a PTT call when all the following conditions are met: The UE has set up a PTT service. There is data to be transmitted for PTT services in the downlink. The value of the IE Cell update cause contained in this message is paging response.
