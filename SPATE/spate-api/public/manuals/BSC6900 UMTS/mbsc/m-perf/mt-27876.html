67194095 VS.SCTP.IPLAYER.RXMEANSPEED T7268:Average Receive Rate of an SCTP Link IP Layer

Description

These counters provide the receive rate or transmit rate of an SCTP link IP layer in the specified measurement period. The rates measured here include the spending of the IP header and the payload of the IP packet.

Measurement point

Samples are taken every 5 seconds in the specified measurement period. The system obtains the number of bytes received and sent on the link once every 5 seconds. With this, the system obtains the maximum and the minimum number of bytes received or sent in 5 seconds, and the total number of bytes in one measurement period.

A Stream Control Transmission Protocol (SCTP) link can be configured for one or multiple boards as follows:
  • A standalone board
  • Two standalone boards
  • One or two pairs of active and standby boards
  • A pair of active and standby boards and a standalone board
If an SCTP link is configured for multiple boards, the system calculates the number of bytes received from and sent to each of the boards every 5 seconds and compares to obtain the maximum and minimum numbers of bytes received from and sent to each board every 5 seconds. Then the system compares the maximum and minimum numbers obtained for all the boards to obtain the maximum and minimum numbers of bytes received and sent on the SCTP link every 5 seconds. The maximum and minimum speed is calculated with the maximum and minimum bytes in 5 seconds, and the average speed is calculated with the total bytes in one measurement period.
