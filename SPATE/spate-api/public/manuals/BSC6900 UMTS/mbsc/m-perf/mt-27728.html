Compression Ratio of DL Packets Under ROHC

Measurement Counters

ID Counter Description
67204485 VS.PDCP.DL.ROHC.PktCompressRatio Compression Rate of DL Packets After ROHC Compression

Description

This measurement item takes statistics of the ratio of the packet size after all downlink PS packets undergo Robust Header Compression (ROHC) to the packet size before the ROHC during a measurement period in the RNC. This measurement item is valid when the ROHC algorithm is configured.

Measurement point

When a PDCP entity receives a downlink packet, it judges whether the packet is configured with ROHC. If the packet is not configured with ROHC, the PDCP entity does not report this measurement item. If the packet is configured with ROHC, the number of compressed downlink packets increases by one. Meanwhile, the PDCP entity adds the initial packet size to the item DL Packet Size before Compression. After the compression, it adds the packet size to the item DL Packet Size after Compression. At the end of the measurement period, the PDCP entity reports the two item values to the performance measurement unit and resets the two items. Then the performance measurement unit calculates the compression ratio of the downlink packets under ROHC based on the reported values of DL Packet Size before Compression and DL Packet Size after Compression. Both service data and packet headers are measured.

Formula

None

Unit

%
