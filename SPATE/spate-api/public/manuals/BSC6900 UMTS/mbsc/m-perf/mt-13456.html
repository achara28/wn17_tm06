Number of RRC Connection Requests of Different Priorities Discarded During RAN Integrated Flow Control for Cell

73393777 VS.NormPriRRC.RanFC.Disc.Num Number of Middle-Priority RRC Connection Setup Requests Discarded During RAN Integrated Flow Control for Cell
73393778 VS.HighPriRRC.RanFC.Disc.Num Number of High-Priority RRC Connection Setup Requests Discarded During RAN Integrated Flow Control for Cell

Description

These counters provide the number of RRC connection requests of different priorities discarded in a cell in phase 2 of RAN integrated flow control. In this phase, the RNC performs RAN integrated flow control based on NodeB-reported CPU load information.

Measurement point

After receiving an RRC CONNECTION REQUEST message from a UE, the RNC determines whether to set up the RRC connection based on the flow control algorithm if RAN integrated flow control is being performed in the cell.

If the RNC determines that the RRC connection will not be set up, the subsequent procedure is as follows:

If the current number of RRC connection rejections does not exceed the configured RRC connection rejection threshold, the RNC responds to the UE with an RRC CONNECTION REJECT message.

Otherwise, the RNC discards the RRC CONNECTION REQUEST message directly and measures the number of discarded RRC connection requests.
