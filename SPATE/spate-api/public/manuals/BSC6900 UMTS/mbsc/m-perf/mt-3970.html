Description

The measurement items provide the number of failed radio link additions for different causes on the Iur interface for SRNC.

The following table describes the meaning of each counter.

Counter Description
VS.SHO.FailRLAddIur.HW.Rx Number of failed radio link additions caused by hardware failure on Iur interface for SRNC (cause value: Hardware Failure)
VS.SHO.FailRLAddIur.TransCong.Rx Number of failed radio link additions caused by insufficient RNC transmission capability on Iur interface for SRNC (cause value: Transport Resource Unavailable)
VS.SHO.FailRLAddIur.Cong.Rx Number of failed radio link additions caused by insufficient RNC capability on Iur interface for SRNC (cause value: Cell not Available, UL Scrambling Code Already in Use, DL Radio Resources not Available, UL Radio Resources not Available, Combining Resources not Available, Measurement Temporarily not Available, Cell Reserved for Operator Use, Control Processing Overload, Not enough User Plane Processing Resources or Transport Resource Unavailable)
VS.SHO.FailRLAddIur.CfgUnsupp.Rx Number of failed radio link additions caused by improper configurations on Iur interface for SRNC (cause value: Combining not Supported, UL Spreading Factor not Supported, DL Spreading Factor not Supported, Requested Tx Diversity Mode not supported, or CM not Supported)
VS.SHO.FailRLAddIur.NodeBReHost.Rx Number of failed radio link additions caused by NodeB Rehoming on Iur interface for SRNC (cause value: NodeB Rehoming)

Measurement point

The measurement is triggered at point B, as shown in the figure. When the SRNC receives a RADIO LINK ADDITION FAILURE message from the DRNC, the SRNC measures the item according to different failure causes on each Iur interface.
